# 歌詞落下リズムゲーム デザイン

## 概要

TyranoScript サンプルの `docs/` ルートに、新しいブラウザ向けリズムゲームを追加する。プレイヤーは `maou_short_14_shining_star.mp3` に合わせて、上から落ちてくる歌詞の1文字を `A` `S` `D` `F` の4レーンで処理する。

## 目的

- 既存の TyranoScript プロジェクトを大きく壊さずに、新作ミニゲームを追加する
- 楽曲と歌詞を使った、短時間で遊べる反応型ゲームを作る
- シナリオ制御とリアルタイムゲーム制御の役割分担を明確にする

## プレイヤー体験

プレイヤーは開始前に「音が出る」ことを大きく告知され、明示的に開始操作をしてからゲームを始める。ゲーム中は歌詞の1文字が上から落ちてきて、文字が判定枠に入るタイミングで対応レーンのキーを押す。成功するとスコアとコンボが伸び、ミスすると減点されてライフが減る。

## ゲームループ

1. タイトル画面を表示
2. 音が出ることを明示した開始前オーバーレイを表示
3. プレイヤーが開始ボタンを押すと楽曲再生と譜面進行を開始
4. 手動タイミング表に従って歌詞1文字を順番に出現させる
5. 各文字をランダムな `A/S/D/F` レーンへ割り当てて落下させる
6. 判定枠で正しいキーを押せば加点、ミスや取り逃しで減点とライフ減少
7. ライフが 0 になったら即ゲームオーバー
8. ライフが残ったまま曲が終わったらクリア
9. 結果画面からリトライできる

## 画面構成

### タイトル

- ゲームタイトル
- 簡単なルール説明
- 開始導線

### 開始前オーバーレイ

- 「音が出ます」の注意書きを大きく表示
- 音量確認を促す補助文
- `開始する` ボタン

### プレイ画面

- 上部: スコア、コンボ、残りライフ
- 中央: 4レーンと落下中の歌詞文字
- 下部: 判定枠と `A S D F` のキー表示
- 補助表示: 曲名

### 結果画面

- `CLEAR` または `GAME OVER`
- 最終スコア
- 最大コンボ
- リトライ導線

## 実装方針

`first.ks` はタイトル、注意書き、ゲーム開始トリガー、結果表示の制御に絞る。リアルタイム部分は JavaScript で `#tyrano_base` 上へ専用 UI を描画し、TyranoScript はミニゲーム開始と終了の橋渡しだけを担当する。

実装は DOM ベースで行い、`requestAnimationFrame` でノーツ位置を更新する。初版は 4 レーン固定、1文字ノーツのみ、手動譜面データ固定で進める。

## データ設計

譜面データは JavaScript 配列で管理し、各ノーツは以下の情報を持つ。

- `time`: 曲開始からの判定時刻
- `char`: 表示する歌詞1文字
- `lane`: 0 から 3 のレーン番号
- `hit`: 判定済みかどうか

歌詞文字列とタイミング表は固定データとして保持し、ゲーム開始時に結合して譜面配列を作る。レーンは開始時にランダム割当するが、1度決まったレーンはそのプレイ中では固定する。

## 判定仕様

- 判定キーは `A` `S` `D` `F`
- レーンとキーの対応は固定
- 判定幅は初版で `±120ms` を成功とする
- 成功時はスコア加算、コンボ継続
- ミス、別キー入力、取り逃しは減点、コンボ切断、ライフ減少
- 初期ライフは 10
- ライフが 0 で即終了

## ファイル構成

- `docs/index.html`: CSS と JavaScript の追加読み込み
- `docs/data/scenario/first.ks`: タイトル、注意書き、開始導線、結果表示
- `docs/data/system/rhythm_game_data.js`: 歌詞とタイミング表
- `docs/data/system/rhythm_game_core.js`: 判定、スコア、ライフ、終了判定
- `docs/data/system/rhythm_game_ui.js`: DOM 描画、入力、音声同期
- `docs/data/system/rhythm_game.css`: レーン、判定枠、注意書き、結果画面の見た目

## テスト方針

TDD で進める。まず DOM に依存しないコアロジックからテストする。

- 譜面生成で文字数とタイミング数が正しく結合される
- ランダムレーン割当が `0..3` に収まる
- 正しいキー入力でスコア加算される
- 別キー入力や取り逃しでライフが減る
- ライフ 0 でゲームオーバー判定になる
- 曲終了時にライフが残っていればクリア判定になる

ブラウザでは以下を手動確認する。

- 開始前に音量注意オーバーレイが目立つ形で表示される
- `開始する` を押すまで音が鳴らない
- `A/S/D/F` 入力で判定が動く
- 10 回ミスでゲームオーバーになる
- 曲完走でクリア結果が表示される

## 初版でやらないこと

- 難易度選択
- 長押しノーツ
- スコアランキング
- 自動譜面解析
- 複雑な演出分岐
